Monthly auto update on local pc
	get_service_date = 'auto'
	python_processing = 'local_pc'

=============================================================================================

0. Set up a PC for local production with Transit Time Map. see instructions document "Set up a PC for local production with Transit Time Map"

1. Run Docker Desktop for Windows

2. Go to the folder where you cloned the Navitia-docker-compose repo e.g. \Dev\navitia-docker-compose

3. Right-click and select run "Git Bash" and type at the prompt: 
	$ docker-compose up

4. Update the config file - \root\transitanalystisrael_config.py 
	a. remove "#" from the "Monthly auto update on local pc" to un-comment parameters to look as below:

#Monthly auto update on local pc
get_service_date = 'auto'
python_processing = 'local_pc'
ttm_graph_processing = 'local_pc'
web_client_hosted_on = 'local_pc'
ttm_server_on = 'local_pc'

	b. make sure all the rest of the products are commented out - have "#"in the first char of the line.

	c. Remeber that GTFS is uploaded to MOT servers at 10pm, so gtfsdate date should be as in the MOT site ftp://gtfs.mot.gov.il

This product uses transit time map:
-----------------------------------
b. Set in the config file navitia_docker_compose_file_path where you downlaoded navitia-docker-compose folder, e.g.: "..\Dev\\Nativia\\navitia-docker-
compose"
c. Set time_map_server_url to "http://localhost:9191/"

5. Run transitanalystisrael_v1.py with python3
